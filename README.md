## Что реализовано
### API-клиент

Для работы с API создан отдельный класс ```EntityApi```.

Сейчас реализован метод:

- ```create_entity``` — отправляет POST-запрос на создание сущности
Модели данных

Для формирования тела запроса используются Pydantic-модели:

- ```AdditionRequest```
- ```EntityRequest```

Payload для теста собирается через модель ```EntityRequest```, после чего преобразуется в словарь через ```model_dump()```.
Это позволяет не собирать тело запроса вручную в тесте и держать структуру данных отдельно от тестовой логики.

## Автотесты

На текущем этапе реализован тест:

- создание сущности через ```POST /api/create```

Проверяется:

- что API возвращает статус-код 200
- что тело ответа не пустое
- что в ответе возвращается id созданной сущности
- что id является числом

## Покрытие по заданию

По заданию требуется создать проект из пяти API-автотестов для всех точек доступа к приложению.

На текущем этапе реализован первый положительный API-тест:


### 1. Создание сущности

Проверяется сценарий:

1. Сформировать тело запроса для создания сущности.
2. Отправить POST-запрос на ```/api/create```.
3. Проверить, что сервер вернул статус-код 200.
4. Проверить, что в ответе вернулся id созданной сущности.
5. Проверить, что id является числом.

Остальные API-тесты будут добавляться по мере реализации покрытия остальных точек доступа приложения.

## Установка и запуск
### 1. Клонировать репозиторий
```commandline
git clone https://github.com/AmirRoni/SimbirSoft-SDET-Week-2.git
cd SimbirSoft-SDET-Week-2
```
### 2. Создать и активировать виртуальное окружение

Windows:
```commandline
python -m venv .venv
.venv\Scripts\activate
```
macOS / Linux:
```commandline
python3 -m venv .venv
source .venv/bin/activate
```
### 3. Установить зависимости
```commandline
pip install -r requirements.txt
```
### 4. Запустить тестовое приложение
Перед запуском тестов приложение должно быть доступно по адресу:
```commandline
http://localhost:8080
```
5. Запустить все тесты
```commandline
pytest
```
